
Как отказаться от изменений в файле (файлах) находящихся в рабочем каталоге?

Используя команду git checkout -- <file> для отмены изменений в конкретном файле.  git checkout -- . для отмены всех изменений в рабочем каталоге.

Как вернуть файл из индекса в рабочий каталог?

Команда git reset HEAD <file> для удаления файла из индекса и возврата его в рабочий каталог.

Как отменить ошибочный commit?

Команда git reset --soft HEAD~1, чтобы отменить последний commit, оставив изменения в рабочем каталоге. Можно использовать git reset --hard HEAD~1, чтобы отменить последний commit и удалить изменения из рабочего каталога.

Как исправить commit, отправленный на сервер GitHub?

Команда git commit --amend для исправления последнего commit, затем  git push --force для отправки исправленного коммита на сервер GitHub. Однако использование  --force, может повлечь потерю истории изменений.

Как создать новую ветку и переключиться в нее?

Команда git checkout -b <branch_name> для создания новой ветки и переключения в нее одновременно. Если ветка существует, команда git checkout <branch_name> для переключения на существующую ветку.

Как создать git репозиторий из локального каталога на своем компьютере и отправить его на github.com?

Можно инициализировать локальный репозиторий с помощью команды git init, затем добавить файлы в репозиторий с помощью git add <file> и закоммитить изменения с помощью git commit -m "Initial commit". Затем создать пустой репозиторий на GitHub, скопируйте URL репозитория, и использовать git remote add origin <URL> для связи локального и удаленного репозиториев. Ну и в конце git push -u origin master для отправки изменений на GitHub.

Как получить на локальный компьютер изменения с github.com?

Команда git pull origin <branch_name> для получения изменений с удаленного репозитория GitHub и их объединения с текущей веткой на вашем локальном компьютере.

Какую информацию нам дают команды:

git status: Показывает текущее состояние  рабочего каталога и индекса, например, измененные файлы, неотслеживаемые файлы, и т.д.

git log: Показывает историю коммитов, список выполненных коммитов с их идентификаторами, авторами, датами и сообщениями коммитов.

git remote -v: Показывает список удаленных репозиториев и их URL-адресов, связанных с  локальным репозиторием.

git branch -v: Показывает список веток в  локальном репозитории и последних коммитов на каждой ветке.
